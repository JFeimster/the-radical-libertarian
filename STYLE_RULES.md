# Style Rules

## Writing

Use:

- Short paragraphs
- Strong H2/H3 sections
- Concrete examples
- Sharp metaphors
- Clear CTAs
- Memorable one-liners

Avoid:

- Beige filler
- Corporate hedging
- Academic fog
- Fake urgency
- Unsupported claims
- Ten-dollar words doing five-cent labor

## HTML

Use semantic HTML:

- `header`
- `nav`
- `main`
- `section`
- `article`
- `footer`

Every page should include:

- Title
- Meta description
- Header/nav
- Page hero
- Main content
- Footer
- Script reference

## CSS

Reuse existing classes before creating new ones.

Preferred patterns:

- `page-hero`
- `sticker`
- `btn`
- `site-header`
- `site-nav`
- `site-footer`
- `resource-tile`
- `tool-tile`
- `seo-card`

## JSON

Use consistent fields:

- `title`
- `slug`
- `url`
- `category`
- `tags`
- `excerpt`
- `description`
- `status`
- `notes`

## Status Values

- `planned`
- `draft`
- `review`
- `active`
- `published`
- `archived`
- `needs-url`
- `needs-review`
- `external-resource`
